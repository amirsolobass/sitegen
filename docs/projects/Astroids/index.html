Astroids

A fully playable Asteroids clone built with Pygame — my first game. Features procedurally spawning asteroid fields, player shooting, collision detection, a shield mechanic, power-ups, a high score system, and a full menu and end screen.

Features

  • Player movement: rotation-based thrust with momentum
  • Asteroid splitting: large asteroids split into smaller fragments on impact
  • Asteroid field: continuous procedural spawning that escalates over time
  • Shooting: projectile physics with cooldown
  • Shield: protective mechanic with limited duration
  • Power-ups: pickups with varied effects (infrastructure in place)
  • High score system: best score persisted to disk across sessions
  • Menu and end screen: full game loop with restart flow
  • Event logging: game events and state written to JSONL for replay or analysis

How It Works

The game is built around Pygame's sprite system. All entities — the player, asteroids, shots, and power-ups — extend a base CircleShape class that handles circle-based collision detection via radius comparison.

The AsteroidField spawner selects a random edge of the screen each tick and launches an asteroid at a random angle toward the playfield. On destruction, an asteroid spawns two or three smaller children, creating the characteristic splitting cascade.

Game events (player death, asteroid splits, power-up pickups) are logged to game_events.jsonl, and a full state snapshot is written to game_state.jsonl at regular intervals — useful for debugging or potential replay functionality.
